# CLAUDE.md

## 基本ルール

- **応答・ファイル記述はすべて日本語**で行う
- コミット・プッシュなど破壊的操作は必ずユーザーの確認を取ってから実行する
- 実装前に方針をユーザーに提示してレビューを受ける
- `git add .` / `git add -A` は使わず、対象ファイルを個別にステージする

## 最優先ドキュメント

**AGENTS.md** を常に遵守すること。他のドキュメントと矛盾する場合は AGENTS.md が優先。

## スラッシュコマンド

| コマンド | 内容 |
|---------|------|
| `/start` | セッション開始・必読ドキュメントの確認 |
| `/refresh` | コンテキストリフレッシュ（中断後の状態把握） |
| `/plan` | 実装計画の立案・ユーザーレビュー |
| `/commit` | 変更のコミット・プッシュ |
| `/update-docs` | ROADMAP / CHANGELOG / SESSION.md の更新 |

## セッション名

- セッション開始時（`/start` 実行後）に、作業内容を日本語でセッション名として提案する
- ユーザーが承認したら `/rename` で設定する
- 形式例：`Phase29 漫画モード実装`、`修正 サムネイル取得バグ`、`整理 any型除去`

## コミット形式

```
[種別] 変更内容の要約（日本語・動詞で締める）

- 変更点1
- 変更点2
```

種別: `[機能追加]` `[改善]` `[修正]` `[整理]`
