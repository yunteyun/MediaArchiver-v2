# コミット手順

AGENTS.md のコミット規約に従ってコミットを作成します。

## 1. 変更内容を確認

```bash
git status
git diff
```

## 2. 対象ファイルを個別にステージング

`git add .` や `git add -A` は使わず、対象ファイルを明示してステージする。

```bash
git add src/components/Foo.tsx electron/ipc/bar.ts
```

> 機密ファイル（.env など）を誤って含めないようにするため、ワイルドカードでの一括ステージングは避けること。

## 3. コミットメッセージを作成（AGENTS.md の形式に従う）

```
[種別] 変更内容の要約（日本語・動詞で締める）

- 変更点1
- 変更点2
```

**種別の定義:**
- `[機能追加]` 新しい機能の追加
- `[改善]` 既存機能の改善・調整
- `[修正]` バグ・不具合の修正
- `[整理]` リファクタリング・ドキュメント整理（動作変更なし）

**OK例:**
```
[改善] FileCard 各モードの情報欄レイアウトを調整

- バッジ化後に情報が詰まって見える問題を改善
- タグチップの幅上限と表示文字量を拡張
```

## 4. コミット

```bash
git commit -m "$(cat <<'EOF'
[種別] 要約

- 変更点
EOF
)"
```

## 5. プッシュ前の確認

- CHANGELOG と ROADMAP の更新がコミットに含まれているか
- 意図しないファイルがステージングに含まれていないか（`git status` で確認）

```bash
git push origin main
```

## リリース時の必須手順

新しいリリースを行う場合は以下を**必ず**更新すること：

1. **`package.json` の `version` フィールド** を新しいバージョン番号に変更
   - `app.getVersion()` 経由でヘッダーと設定モーダルに表示されるため、ここを変えないとバージョン表記が古いまま
   - バージョン規則: 機能追加 → `MINOR` を上げる、修正のみ → `PATCH` を上げる

2. **`CHANGELOG.md`** の `[Unreleased]` を整理してバージョンセクションへ昇格

3. **リリースビルドコマンド**（順番通りに実行）:
   ```bash
   npm run build
   npm run build:electron
   ```
   - 出力先: `release/` フォルダ内に ZIP が生成される

## ユーザー向けアップデート方法

> 設定 → サムネイルタブ → 「保存場所」で現在のモードを確認してからアップデートすること。

### AppData モード（デフォルト）/ カスタムパスモード
データは `%APPDATA%\MediaArchiver v2\` に保存されるため、インストールフォルダはまるごと削除して新しい ZIP を展開してOK。

1. アプリを終了
2. インストールフォルダをまるごと削除
3. 新しい ZIP を展開

### ポータブルモード（インストールフォルダ内に保存）
インストールフォルダ内の `data\` にデータが入っているため退避が必要。

1. アプリを終了
2. `data\` フォルダを別の場所に退避
3. インストールフォルダをまるごと削除
4. 新しい ZIP を展開
5. `data\` フォルダを元の場所に戻す
