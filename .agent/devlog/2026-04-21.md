## 実装した機能

### フェーズ D-3: ホバー挙動のプリセット駆動化

**背景**: D-2 で実装した WhiteBrowser マウス位置連動コマ送りが `useFileCardHover.ts` の `displayMode === 'whiteBrowser'` ハードコードに依存していた。外部プリセットで同等の挙動を宣言できない設計上の問題があった。

**対応**: `HoverBehavior` 型を追加し、プリセット定義に `hoverBehavior?: HoverBehavior` フィールドを組み込んだ。`useFileCardHover` の引数を `displayMode` から `hoverBehavior` に置換し、`whiteBrowser.ts` に `hoverBehavior: 'mousePositionScrub'` を明示。表示への変更なし。

**コミット**: `3f0b368`

### フェーズ D-4: archive 並列制御 + プレビューフレーム数 UI 改善

**背景**: 調査により archive プレビュー取得に 2 つの問題を発見。(1) `getArchivePreviewFrames` に in-flight coalesce が無く、同一ファイルへの並行呼び出しで 7za が多重起動する。(2) 7za 抽出ループに pLimit が無く、複数ファイル並行で spawn が爆発する。また video プレビューフレーム数設定のスライダが値の意味を伝えにくかった。

**対応**:
- `previewLimit = pLimit(1)` を新設し 7za 抽出を直列化
- `previewInflight` Map で in-flight coalesce を実装（`getArchiveMetadata` と同パターン）
- プレビューフレーム数 UI をスライダ → ラジオ 4 択（オフ / 低5 / 中10 / 高20）に変更

**コミット**: `c4226ab`

### テスト修正: 既存 7 件失敗の全解消

**背景**: 以前から 3 ファイル 7 件のテスト失敗が放置されていた。調査の結果、実装変更時のテスト追従漏れ 2 件と、アルゴリズムの実際のバグ 1 件に分類できた。

**内容**:
- **`duplicateNameCandidates.ts` バグ修正**: `seriesBase === compactName`（末尾に数字がない）時も `numbered_series` キーを生成していた。「Room 1408 Director Cut」のようなタイトル中の数字を連番と誤判定し、無関係ファイルが重複候補に混入する問題。条件に `&& seriesBase !== compactName` を追加して修正。
- **`useDuplicateStore.test.ts`**: `findDuplicates` に `folderIds` 引数が追加された際にテストの `toHaveBeenCalledWith` が未更新だった。`(mode, undefined)` に修正。
- **`profileLifecycle.test.ts`**: `showDriveBadge` / `driveColors` / `infoBadgeOrder` / `showCreatedDate` / `showFolderBadge` がプロファイル設定に追加された際、テストモックと期待値が未追従だった。各テストケースに不足フィールドを追加。

**結果**: 全 124 件グリーン。

**コミット**: `5662a63`
