# update.bat の使い方（忘れた時用）
1. 新しいリリースZIPをダウンロードする
2. アプリを置いているフォルダを開く
3. 同じフォルダにある `update.bat` をダブルクリックする
4. ZIP選択ダイアログが出たら、ダウンロードしたZIPを選ぶ
5. 自動で「アプリ終了 → 上書き更新 → 再起動」される
6. 再起動後、ヘッダーのバージョン表示と通常操作を確認する

## 注意
- `update.bat` は `MediaArchiver v2.exe` と同じフォルダで実行する
- 更新中はアプリを手動で操作しない
- 失敗したら、もう一度 `update.bat` を実行する（ZIPを選び直す）
- 2026-02-25 修正版では、ZIP更新後に起きることがあった `". was unexpected at this time."` エラーを修正済み

## 更新後の最小確認（おすすめ）
- アプリが起動する
- ヘッダーのバージョン表示が想定どおり
- フォルダ一覧が表示される
- ファイル一覧が表示される（サムネイルが1件でも出る）
- ログに起動時エラーが大量に出ていない

## アプリ内での更新確認
- `設定 > 更新` から最新版の有無を確認できます。
- `設定 > 更新` では、現在インストール中の版に同梱されたリリースノートも確認できます。
- 更新確認結果には、最新リリースの更新内容（リリースノート）が表示されます。
- 更新がある場合は `検証済み更新ZIPを取得` でZIPをダウンロードできます。
- `検証済み更新ZIPを取得` は `.sha256` での検証が前提です。ハッシュ検証できないZIPは取得失敗になります。
- ZIP取得後は `update.bat で適用` から更新適用を起動できます（実行時にアプリは終了します）。
- `リリースページを開く` から GitHub Releases を直接開き、手動で ZIP を取得することもできます。
- `ZIPを選んで適用（従来方式）` では、`update.bat` のZIP選択ダイアログを使って手元のZIPを指定できます。

## 失敗時の切り分け（簡易）
- ZIP選択直後に失敗: ZIPファイル破損 / 展開権限 / パス問題を疑う
- 更新完了後に起動しない: ウイルス対策ソフトの隔離・ロックを疑う
- 旧版のままに見える: 別フォルダのアプリを起動していないか確認する
- 何度も失敗する: 新しいZIPを再ダウンロードして再実行する

## 表示モードの意味（レイアウト / サムネイル）

### レイアウト（表示）
- `標準（S/簡易）`: 情報欄を最小にした高密度表示
- `標準（M）`: バランス型の標準表示
- `標準（L）`: 標準（M）より大きめ表示
- `縦型`: 縦長サムネイル向け表示
- `動画（ワイド）`: 横長サムネイル向け表示
- `詳細表示`: `サムネイル左 / 情報右` の横長カード表示
- `詳細表示（縦）`: `サムネイル左（縦長） / 情報右` の横長カード表示

### サムネイル表示方式
- `モード既定`: レイアウトごとの既定表示に従う
- `切り取り（cover）`: 枠いっぱいに表示（はみ出し部分は切り取り）
- `全体表示（contain）`: 画像全体を表示（余白が出ることがある）
- `固定（square）`: 正方形サムネイル互換向けの設定値

### 補足（2026-03-06 時点）
- 現行実装では `切り取り（cover）` と `固定（square）` は見た目の差がほぼありません。
- `固定（square）` は拡張用の予約値として維持しており、主に `詳細表示` 互換の意味で扱っています。
