# 保存場所とログ

MediaArchiver v2 は、用途によって保存場所を分けています。

## まず知っておきたいこと

- 普段使うデータは、設定した保存先に置かれます
- 起動直後のログだけは、起動トラブル調査のため `AppData` 側にも残ります
- 保存先を変更しても、`storage-config.json` は起動のために `AppData` 側へ残ります

## 何がどこに保存されるか

| 種類 | 主な場所 | 補足 |
| --- | --- | --- |
| 起動用の設定 | `C:\Users\<ユーザー名>\AppData\Roaming\media-archiver-v2\storage-config.json` | アプリが「本当の保存先」を知るための最小設定です |
| 起動直後のログ | `C:\Users\<ユーザー名>\AppData\Roaming\media-archiver-v2\bootstrap-logs\release` | 起動しない時の確認用です |
| 開発版の起動直後ログ | `C:\Users\<ユーザー名>\AppData\Roaming\media-archiver-v2\bootstrap-logs\dev` | 開発版とリリース版が混ざらないように分けています |
| 通常ログ | `<現在の保存先>\logs` | 起動後の通常動作ログです |
| データベース | `<現在の保存先>\*.db` | プロファイルやファイル情報が入ります |
| サムネイル / プレビュー | `<現在の保存先>\thumbnails` | 画像・動画・書庫のサムネイルです |
| バックアップ設定 | `<現在の保存先>\backup-settings.json` | バックアップ先や世代数の設定です |
| 外部表示プリセット | `<現在の保存先>\display-presets` | 外部 JSON の表示プリセットです |
| バックアップ本体 | `<現在の保存先>\backups` | 別の保存先を指定している場合はそちらが優先されます |

## 保存先モードごとの例

### AppData モード

- 通常データ: `C:\Users\<ユーザー名>\AppData\Roaming\media-archiver-v2\release`
- 通常ログ: `C:\Users\<ユーザー名>\AppData\Roaming\media-archiver-v2\release\logs`

### インストールフォルダ モード

- 通常データ: `MediaArchiver v2.exe` と同じ場所の `data`
- 例: `D:\Program\MediaArchiver v2\data`

### 任意の場所 モード

- 設定で選んだフォルダがそのまま保存先になります

## 起動しない時はどこを見るか

1. まず `bootstrap-logs\release` の今日の日付のログを確認する
2. 起動できる場合は、次に現在の保存先の `logs` も確認する
3. 更新直後の不具合なら、保存先より先に `bootstrap-logs` を見る方が早いことがあります

## 補足

- 以前の版で `AppData` 側に保存されていたバックアップ設定や外部表示プリセットは、必要に応じて新しい保存先へ自動で引き継がれます
- `AppData` 側の起動ログは、調査のために残す想定です。通常は手動削除しなくて大丈夫です
