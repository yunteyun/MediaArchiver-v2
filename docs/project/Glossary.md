# Glossary (用語集)

MediaArchiver v2 で使う用語の定義です。画面名、設定名、設計文書内の呼び方を揃えるために使います。

---

## アプリケーション概念

### プロファイル (Profile)
ユーザーが管理するライブラリの単位。プロファイルごとにデータベース、登録フォルダ、タグ、スマートフォルダ、スキャン系設定を分離する。

### メディアファイル (MediaFile)
アプリケーションの管理対象ファイル。画像、動画、書庫、音声を含む。

### 登録フォルダ
ライブラリとして監視・スキャン対象に追加したフォルダ。サイドバーから選択できる。

### スキャン
登録フォルダを走査し、ファイル情報、メタデータ、サムネイル関連情報を取り込む処理。

### スマートフォルダ (Smart Folder)
現在の絞り込み条件を名前付きで保存し、あとから再適用できる機能。v1 では保存検索型を中心に扱う。

### 保存モード
アプリのデータ保存先の扱い方。`install` モードではアプリフォルダ配下の `data` を使う。

---

## ファイル種別

### 画像ファイル
jpg, jpeg, png, gif, webp, bmp, tiff などの静止画やアニメーション画像。

### 動画ファイル
mp4, mkv, avi, mov, webm, wmv, flv などの映像ファイル。

### 書庫 / アーカイブ (Archive)
zip, cbz, rar, cbr, 7z などの圧縮ファイル。画像主体の書庫は一覧表示やプレビューの対象になる。

### 音声ファイル
mp3, wav, m4a, flac, ogg, aac などの音声ファイル。

---

## UI 用語

### ファイルカード (FileCard)
一覧グリッドに表示される個々のアイテム。サムネイル、タイトル、タグ要約、サイズや作成日などの情報を持つ。

### 一覧ヘッダー
検索、並び替え、グループ化、表示切替、ファイルタイプ絞り込みなどをまとめた上部操作領域。

### サイドバー (Sidebar)
画面左側のナビゲーション領域。フォルダ、タグ、評価、スマートフォルダなどの選択に使う。

### 右パネル (Right Panel)
選択中ファイルの情報表示と編集を行う右側領域。タグ、評価、メモなどを扱う。

### Center Viewer
中央カラム内だけで完結するメディア表示領域。全画面オーバーレイは採用せず、右パネルと責務を分ける。

### ライトボックス (Lightbox)
旧称として残る呼び名。現行文脈では `Center Viewer` を指すことが多いが、過去文書では旧実装の全画面寄りプレビューを指す場合がある。

### クイックプレビュー (Quick Preview / Quick Look)
カード上や補助 UI から、その場で軽く内容確認するための小さなプレビュー。

---

## 表示・レイアウト

### レイアウトプリセット (Layout Preset)
カード構造や寸法を決める軸。例: `compact` `standard` `standardLarge` `manga` `video` `detailed`。

### サムネイル表示方式 (Thumbnail Presentation)
サムネイルの見せ方を決める軸。例: `modeDefault` `cover` `contain` `square`。

### 表示プリセット (Display Preset)
レイアウト、情報欄、タグ要約 UI、サムネイル表示方式などをまとめた設定単位。built-in に加え、外部 JSON から追加できる。

### 標準（S/M/L）
標準系のカード表示。`S` は高密度、`M` はバランス型、`L` はやや大きめ表示。

### 縦型
縦長サムネイル前提で見やすくした表示。旧文脈の `漫画` モードに近い。

### 動画（ワイド）
横長サムネイル前提の表示。動画向けに情報密度や見え方を調整する。

### 詳細表示
`サムネイル左 / 情報右` の横長カード表示。

### 詳細表示（縦）
`サムネイル左（縦長） / 情報右` の横長カード表示。

### タグ要約
カード内に表示するタグの省略表示。件数やチップサイズは表示プリセット側で調整できる。

### サムネイルアクション (Thumbnail Action)
カードに対するホバー時挙動。`scrub` `flipbook` `play` などを含む。

---

## 検索・整理

### タグ (Tag)
ファイルへ付与するラベル。1 ファイルに複数付与できる。

### タグカテゴリ (Tag Category)
タグをまとめる上位概念。色や優先順位を持つ。

### 親タグ / 子タグ
タグの階層関係。例: `キャラクター > 〇〇`。

### 評価 (Rating)
軸ごとに持てる数値評価。絞り込み条件や情報表示に使う。

### ファイルタイプ絞り込み
動画、画像、書庫、音声の種別で一覧対象を絞る機能。

### 検索条件
検索語、タグ条件、評価条件、ファイルタイプ条件、フォルダ選択など、一覧を絞り込むための条件全体。

---

## データ・サムネイル

### サムネイル
一覧や詳細で表示する代表画像。画像、動画、書庫、音声で生成方法が異なる。

### プレビューフレーム
主に動画や書庫の追加プレビュー用に生成する複数フレーム画像。

### メタデータ
ファイルから取得する補助情報。例: 動画長さ、画像数、作成日時、拡張情報。

### ハッシュ (Hash)
ファイル内容から計算する識別値。重複判定や変更検知に使う。

---

## 技術用語

### Renderer Process
React UI を描画する Electron 側の画面プロセス。

### Main Process
Electron のメインプロセス。ウィンドウ管理、IPC、DB、スキャン、サムネイル生成などを担当する。

### IPC (Inter-Process Communication)
Renderer と Main の間でやり取りする通信。

### Zustand
Renderer 側の状態管理に使っているストアライブラリ。

### 仮想スクロール (Virtual Scroll)
見えている範囲だけを描画して一覧表示の負荷を抑える技術。

### 再描画 / リレンダリング (Re-render)
状態変化により React コンポーネントが再度描画されること。不要な再描画はパフォーマンス低下につながる。

### Utility Process / Worker
重い処理をメインスレッドから分離する実行単位。ffmpeg 系処理の切り離しで使う文脈が多い。

## AI 呼称

### オプス (Opus)
Claude Opus 系モデルの通称。設計、計画、レビュー文脈で使うことがある。

### ジェミニ (Gemini)
Gemini 系モデルの通称。実装や補助調査の文脈で使うことがある。
