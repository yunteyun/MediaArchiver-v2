# UI/UX 改善案整理 v1

作成日: 2026-03-06

## 目的
- 直近で進める UI/UX 改善を「実装順」と「検証観点」で先に固定し、手戻りを減らす。

## 前提
- まずは一覧画面（FileGrid / FileCard / Header）を対象にする。
- 大規模な情報設計変更ではなく、既存導線の視認性・操作性改善を優先する。

## 優先順位（直近）
1. 一覧カードの情報把握性改善
2. ヘッダー操作の理解しやすさ改善
3. 右サイドバー表示との同時利用時の密度最適化

## 実装順（スライス）

### スライス1: 一覧カードの情報把握性改善
- 対象:
  - 書庫の件数表記（`NNp`）など、カード上で即読できる補助情報の統一
  - バッジの表示ルール（表示条件・優先度）の明文化
- 完了条件:
  - 同種情報は同じ位置・同じ表記ルールで表示される
  - 主要表示モードで可読性を維持できる

### スライス2: ヘッダー操作の理解しやすさ改善
- 対象:
  - 表示（レイアウト）とサムネイル表示方式のラベル・補助説明の見直し
  - 既定値の意味（互換モード）を誤解しない文言調整
- 完了条件:
  - 初見でも各セレクタの責務を区別できる
  - `固定` が予約値である現仕様を UI/ドキュメントの両面で説明できる

### スライス3: 右サイドバー同時利用時の密度最適化
- 対象:
  - 右サイドバー表示時のカード列数・余白・情報密度の最終調整
  - 主要ウィンドウ幅（狭/中/広）での崩れ・過密の抑制
- 完了条件:
  - 標準（S/M/L）と詳細表示で、狭幅時も読める密度を維持できる
  - 余白過多・情報詰まりの偏りが目立たない

## 検証観点
- 画面幅:
  - 狭幅（右サイドバー表示 + 小さめウィンドウ）
  - 中幅（一般的な作業幅）
  - 広幅（横長ウィンドウ）
- モード:
  - `標準（S/M/L）` `漫画` `動画（ワイド）` `詳細表示`
- データ種別:
  - 画像 / 動画 / 音声 / 書庫 / 音声書庫
- 操作:
  - 表示切替直後
  - スクロール中
  - 選択/フォーカス/ホバー状態

## 非対象（この整理で扱わない）
- ライトボックス全面改修
- 自動更新・配布機構
- 大規模な情報アーキテクチャ再設計
