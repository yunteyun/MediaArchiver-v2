# プロファイル独立性 v1

## 目的
- `プロファイル = 別ライブラリ` として感じられる独立性を強める。
- ただしアプリ全体設定まで無差別に分離せず、`全体 / プロファイル / 一時UI` の境界を先に明確にする。
- WhiteBrowser の完全再現ではなく、MediaArchiver に合う独立粒度を段階的に作る。

## 現状整理
- 既にプロファイル別
  - DB
  - 登録フォルダ
  - タグ / 評価 / スマートフォルダ
  - スキャン系設定の一部
  - 自動整理
  - 一覧の既定表示
  - ファイルカード表示の既定
  - 既定で使う外部アプリ
  - ファイル名検索 / 画像検索の検索先
- まだ全体寄り
  - 再生音量 / パフォーマンスモード
  - 外部アプリ定義
  - 保存場所 / 更新 / ログ
- 一時UI
  - 現在の検索語
  - 現在の選択フォルダ
  - 現在の一覧操作状態
  - モーダル開閉や一時プレビュー状態

## 基本方針
- `ライブラリの見え方や整理方針` はプロファイルへ寄せる。
- `アプリ殻としての動作や保守導線` は全体設定に残す。
- `今この画面だけの状態` は保存設定と分離し、一時UIとして扱う。

## スコープ定義

### 全体設定
- 保存場所
- アプリ更新
- ログ
- 再生音量
- パフォーマンスモード
- ライトボックス背景濃度

### プロファイル設定
- スキャン対象カテゴリ
- スキャン速度
- サムネイル生成条件
- 自動整理
- 一覧既定値
  - 並び替え
  - グループ化
  - 既定の検索対象
  - 既定の表示プリセット
  - 既定のサムネイル表示方式
- ファイルカード表示
  - 表示項目
  - タグ表示ルール
- 外部連携のプロファイル設定
  - 既定で使う外部アプリ
  - ファイル名検索 / 画像検索の検索先

### 一時UI
- 現在の検索語と追加検索条件
- 現在のフォルダ選択
- 現在の一覧操作結果
- 右パネル / ビューア / モーダル状態

## Step 1
- 一覧既定値をプロファイル別へ移す。
- ファイルカード表示の既定値をプロファイル別へ移す。
- プロファイル切替時に、一覧表示と検索対象の既定値も同時に切り替える。
- バックアップ / 設定インポートでも同じ profile settings payload を扱う。

## Step 2
- 既定で使う外部アプリをプロファイル別へ移す。
- 検索先設定をプロファイル別へ移す。
- 外部アプリ定義そのものは全体設定として残す。
- 設定バックアップ / インポートでも、外部アプリ定義は全体設定、既定アプリと検索先はプロファイル設定として分けて扱う。

## Step 3
- 一時UIと保存設定の責務をさらに分離する。
- 現在の検索条件・現在の一覧操作・現在の選択状態を `UIStore` の一時状態として扱う。
- プロファイル切替時は、検索語・追加検索条件・ファイルタイプ絞り込み・重複ビュー・モーダル・一時プレビュー状態をリセットする。
- 新しいプロファイルの既定値は、切替後に `UIStore` へ再適用し、前プロファイルの検索対象や一覧状態を持ち越さない。

## 今回やらないこと
- プロファイルごとの完全別プロセス起動
- 同時に複数プロファイルを開く multi-window 化
- スキンやテーマの全面的なプロファイル分離

## 判断基準
- その設定が「ライブラリごとに変えたいか」で判断する。
- 単なるアプリの操作感や保守導線なら全体設定に残す。
- 現在だけ意味があるものは保存対象にしない。
