# プロファイル独立性 v1

## 目的
- `プロファイル = 別ライブラリ` として感じられる独立性を強める。
- ただしアプリ全体設定まで無差別に分離せず、`全体 / プロファイル / 一時UI` の境界を先に明確にする。
- WhiteBrowser の完全再現ではなく、MediaArchiver に合う独立粒度を段階的に作る。

## 現状整理
- 既にプロファイル別
  - DB
  - 登録フォルダ
  - タグ / 評価 / スマートフォルダ
  - スキャン系設定の一部
  - 自動整理
- まだ全体寄り
  - 一覧の既定表示
  - ファイルカード表示の既定
  - 再生音量 / パフォーマンスモード
  - 外部アプリ / 検索先
  - 保存場所 / 更新 / ログ
- 一時UI
  - 現在の検索語
  - 現在の選択フォルダ
  - 現在の一覧操作状態
  - モーダル開閉や一時プレビュー状態

## 基本方針
- `ライブラリの見え方や整理方針` はプロファイルへ寄せる。
- `アプリ殻としての動作や保守導線` は全体設定に残す。
- `今この画面だけの状態` は保存設定と分離し、一時UIとして扱う。

## スコープ定義

### 全体設定
- 保存場所
- アプリ更新
- ログ
- 再生音量
- パフォーマンスモード
- ライトボックス背景濃度

### プロファイル設定
- スキャン対象カテゴリ
- スキャン速度
- サムネイル生成条件
- 自動整理
- 一覧既定値
  - 並び替え
  - グループ化
  - 既定の検索対象
  - 既定の表示プリセット
  - 既定のサムネイル表示方式
- ファイルカード表示
  - 表示項目
  - タグ表示ルール

### 一時UI
- 現在の検索語と追加検索条件
- 現在のフォルダ選択
- 現在の一覧操作結果
- 右パネル / ビューア / モーダル状態

## Step 1
- 一覧既定値をプロファイル別へ移す。
- ファイルカード表示の既定値をプロファイル別へ移す。
- プロファイル切替時に、一覧表示と検索対象の既定値も同時に切り替える。
- バックアップ / 設定インポートでも同じ profile settings payload を扱う。

## Step 2 候補
- 外部アプリ設定のうち「既定で使うアプリ」をプロファイル別にするか再検討する。
- 検索先設定をプロファイル別にするか、全体設定に残すかを整理する。
- 一時UIと保存設定の責務をさらに分離する。

## 今回やらないこと
- プロファイルごとの完全別プロセス起動
- 同時に複数プロファイルを開く multi-window 化
- スキンやテーマの全面的なプロファイル分離

## 判断基準
- その設定が「ライブラリごとに変えたいか」で判断する。
- 単なるアプリの操作感や保守導線なら全体設定に残す。
- 現在だけ意味があるものは保存対象にしない。
