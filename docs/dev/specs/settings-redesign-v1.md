# 設定再設計 v1

作成日: 2026-03-08

## 目的
- 機能追加のたびに増えてきた設定項目を棚卸しし、全体として一貫した情報設計へ整理する。
- `全体設定 / プロファイル別 / フォルダ別 / 一時UI状態` の保存スコープを分離し、どこに置くべき設定かを明確にする。
- 設定画面から運用操作を切り離し、利用者が「何を決める画面か」を理解しやすくする。

## 背景
- 現状は設定項目が段階的に追加されており、同種の設定でも配置先や保存単位が揃っていない。
- 一覧ヘッダーに常用の表示設定があり、設定モーダル側にも関連項目があるため、既定値と一時操作の境界が曖昧になっている。
- `useSettingsStore` と `useUIStore` の両方に近い責務の状態があり、実際には未使用または旧設計の名残になっている項目がある。

## 現状整理

### 現在の設定タブ
- 一般
- サムネイル
- スキャン
- 評価
- ストレージ
- 外部アプリ
- ログ
- バックアップ

### 現在の主な問題
- 設定と運用操作が混在している。
  - 例: 更新確認、ログ閲覧、バックアップ、ストレージ移行。
- 保存スコープが UI 上で明示されていない。
  - 例: `previewFrameCount` はプロファイル別、`thumbnailAction` は全体設定、フォルダ別スキャン設定は別ダイアログ。
- 常用する表示系設定がヘッダーに分散している。
  - 例: 並び替え、グループ化、サムネイル表示方式、表示プリセット。
- ストア責務が混在している。
  - `useSettingsStore`: 永続設定 + 旧設定 + 一部プロファイル文脈。
  - `useUIStore`: 一時UI状態 + 旧 `sortBy / sortOrder`。

### 整理対象として明示する項目
- `useSettingsStore.activeProfileId`
  - `useProfileStore.activeProfileId` と責務が重複している。
- `useSettingsStore.autoScanOnStartup`
  - フォルダ別設定へ寄せた後も残っている旧設定。
- `useSettingsStore.rightPanelVideoMuted`
  - ストアにはあるが、現時点で UI 導線と利用箇所が見えにくい。
- `useSettingsStore.cardLayout`
  - ストアにはあるが、現行 UI/描画では実効性が薄い。
- `useUIStore.sortBy / sortOrder`
  - 現行の一覧制御は `useSettingsStore` 側を使っており、旧状態の可能性が高い。

## 再設計の基本方針

### 1. 設定と運用操作を分離する
- 設定:
  - 既定の振る舞いを決めるもの。
- 運用操作:
  - 今この場で実行するもの。

設定画面に残すもの:
- 表示、再生、スキャン、保存先、連携先など。

設定画面から切り出す候補:
- 更新確認
- ログ閲覧 / コピー
- バックアップ作成 / CSV入出力
- ストレージ移行 / 旧データ削除

### 2. 保存スコープを UI 上で明示する
- 全体設定
  - アプリ全体に共通。
- プロファイル別
  - 現在のプロファイルだけに適用。
- フォルダ別
  - 登録フォルダごとに適用。
- 一時UI状態
  - 次回起動時まで持ち越さない、または設定画面で編集しない。

### 3. 「既定値」と「その場の操作」を分ける
- 設定画面:
  - 既定の表示モード
  - 既定の並び替え
  - 既定のグループ化
  - 既定の検索対象
- ヘッダー:
  - 今この一覧だけに対する一時変更

v1 ではまず「既定値の保存先」を整理し、ヘッダー UI は段階的に役割を縮退させる。

- ライトボックス背景濃度
- 一覧ホバー時の動作
  - スクラブ
  - 自動パラパラ
  - 再生

- Play モード詳細
- 右サイドバー動画プレビュー
  - 方式

### 4. スキャン
- 対応形式カテゴリ
- プレビューフレーム数
- サムネイル解像度
- スキャン速度調整
- フォルダ別スキャン設定への導線
- 将来追加:
  - 除外拡張子
  - 除外フォルダ名

### 5. 保存 / キャッシュ
- 保存先モード
- カスタム保存先

### 6. 連携

## 片付けるべき旧状態

### v1 で削除または移管したいもの
- `useSettingsStore.activeProfileId`
  - `useProfileStore` に一本化。
- `useSettingsStore.autoScanOnStartup`
  - フォルダ別設定移行済みのため削除候補。
- `useUIStore.sortBy / sortOrder`
  - 現在未参照なら削除候補。

### v1 で扱いを決めるもの
- `useSettingsStore.rightPanelVideoMuted`
  - UI へ正式昇格するか、未使用なら削除するか判断する。
- `useSettingsStore.cardLayout`
  - 現行デザインで使わないなら削除。将来使うなら「一覧表示」へ正式配置。

## 画面構成方針

### ナビゲーション
- 上段: 設定
  - 一般
  - 一覧表示
  - プレビュー / 再生
  - スキャン
  - 保存 / キャッシュ
  - 連携
- 下段: 管理
  - 更新
  - ログ
  - バックアップ / 入出力
  - データ移行

### 各セクションの共通ルール
- 冒頭に 1 行説明を置く。
- 各設定にスコープラベルを付ける。
  - `全体`
  - `プロファイル別`
  - `フォルダ別`
- セクション単位で `初期値に戻す` を置く。
- 低頻度の破壊的操作は通常設定と視覚的に分離する。

## 実装フェーズ案

### Step 1
- 設定項目棚卸しと情報設計を文書化する。
- 旧状態と未使用状態を明示する。

### Step 2
- ストア責務を整理する。
  - `useSettingsStore`: 永続設定のみ
  - `useProfileStore` / folder settings: 文脈依存設定
  - `useUIStore`: 一時UI状態のみ

### Step 3
- 設定画面のタブ構成を新 IA に合わせて再編する。
- 「設定」と「管理」を分離する。

### Step 4
- ヘッダーの表示系設定を「現在の一覧の一時変更」として再定義する。
- 既定値変更は設定画面からも操作可能にする。

### Step 5
- 除外ルール、自動クリーンアップ、設定のインポート / エクスポートなど未整備項目を追加する。

### Step 6
- 既存のストレージ診断・バックアップ導線を活かしつつ、自動整理と設定入出力を実装する。

## 実施済み（2026-03-08）
- `useSettingsStore` から `activeProfileId` と `autoScanOnStartup` を削除し、責務外の旧状態を整理した。
- persist `merge` で旧キーを読み飛ばす互換処理を追加し、既存保存データが残っていても新状態へ寄せられるようにした。
- `useUIStore` から未使用の `sortBy / sortOrder` を削除し、一時UI状態の責務に絞った。
- `useSettingsStore` / `useUIStore` の回帰テストを更新し、ストア整理後も既存状態遷移が維持されることを確認した。
- 設定モーダルの左ナビを `設定` と `管理` の2区分に再編し、更新導線を `一般` タブから専用の `更新` タブへ分離した。
- `一般 / 表示`、`プレビュー / 再生`、`連携` など、実際の内容に合うタブ名と説明へ更新した。
- `useUIStore` に「現在の一覧表示」状態を追加し、並び替え・グループ化・表示プリセット・サムネイル表示方式を一時操作として保持するようにした。
- `一般 / 表示` に既定の一覧表示ブロックを追加し、既定値は設定画面、現在の一覧だけの変更はヘッダーという役割で分離した。
- ヘッダーへ「既定値に戻す」を追加し、現在の一覧表示を保存済み既定値へ戻せるようにした。
- `スキャン` タブに全体設定として `除外拡張子` `除外フォルダ名` `隠しフォルダ除外` を追加し、プロファイル別設定と分けて管理できるようにした。
- スキャン除外ルールを共有 helper として定義し、設定保存・Renderer/Main 同期・スキャナ本体の判定が同じ正規化ルールを使うように揃えた。
- スキャナの初回同期と再スキャン時のクリーンアップに除外ルールを反映し、今回から除外した対象が既存登録に残り続けないようにした。
- スキャン除外ルールの回帰テストを追加し、拡張子 / フォルダ名 / 隠しフォルダ判定の正規化と適用条件を確認できるようにした。
- `ストレージ` タブに `起動時の孤立サムネイル自動整理` としきい値設定を追加し、既存の手動診断 / 手動削除とは別に軽い自動保守を選べるようにした。
- 起動時に自動整理設定を読み取り、孤立サムネイル診断結果がしきい値以上のときのみクリーンアップを走らせる導線を追加した。
- `バックアップ` タブに設定の `書き出し / 読み込み` を追加し、全体設定と現在のプロファイル設定を 1 つの JSON として移行できるようにした。
- 設定入出力 helper と回帰テストを追加し、フォーマット互換（version=1）と基本的な往復が崩れないことを確認できるようにした。

## 受け入れ条件
- 各設定に保存スコープが定義されている。
- 設定と運用操作の境界が UI 上で説明可能になっている。
- `useSettingsStore` / `useUIStore` の重複状態の削減対象が明文化されている。
- 新規機能追加時に「どのセクションへ置くか」「どのスコープへ保存するか」を判断できる基準がある。
