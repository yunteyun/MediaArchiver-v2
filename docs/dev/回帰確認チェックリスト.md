# 回帰確認チェックリスト

## このファイルの目的
- UI調整・表示モード調整後の最低限の確認観点を固定化する
- セッションごとの確認漏れを減らす

## 実施タイミング
- `FileCard` / `FileGrid` / `Lightbox` / 表示モード関連を変更したとき
- リリース前の最終確認

## 実施手順（最小）
1. アプリ起動
2. 代表データを含むフォルダを開く（画像 / 動画 / 書庫 / 音声が混在）
3. 表示モードごとに下記チェックを実施
4. 問題があれば `docs/dev/不具合・要望リスト.md` か `CHANGELOG.md` に反映

## 表示モード共通チェック

### 標準（S/簡易）
- [ ] カード高さ・サムネイル比率が崩れない
- [ ] タイトル/要約情報が切れてもレイアウト崩れしない
- [ ] 選択時（枠線・ring）にガタつかない

### 標準（M）
- [ ] 2段目が `サイズ / 作成日 / フォルダ名` バッジで表示される
- [ ] フォルダ名バッジが長い場合に `truncate` される
- [ ] タグ要約が下段・右寄せで表示される
- [ ] タグが少ない/多い（`+N` 発生）時も崩れない

### 標準（L）
- [ ] 標準（M）と同形式（2段目バッジ列 + 下段タグ）になっている
- [ ] 情報欄高さの増加後もカード全体の行揃えが崩れない
- [ ] タグバッジの文字量・高さが標準（M）と概ね揃う

### 漫画
- [ ] 2段目が `サイズ / 作成日 / フォルダ名` バッジで表示される
- [ ] フォルダ名バッジが短め幅で省略表示される
- [ ] タグ要約が下段・右寄せで表示される（件数/幅が控えめ設定）
- [ ] 縦長サムネイルで情報欄が詰まらない

### 動画（ワイド）
- [ ] アスペクト比が崩れない
- [ ] Duration バッジ、拡張子バッジが重ならない
- [ ] 情報欄のメタ行/タグ行が崩れない

## 追加のUI動作チェック（変更時）
- [ ] 表示モード切替直後にグリッドの一時崩れが発生しない
- [ ] スクロール中にカード高さ差で段差が目立たない
- [ ] 選択状態/フォーカス状態/ホバー状態で見た目が破綻しない
- [ ] タグポップオーバー（`+N`）が開閉できる（hover/click 設定の双方を変更時のみ）

## データ種別チェック（最低1件ずつ）
- [ ] 画像
- [ ] 動画
- [ ] 音声
- [ ] 書庫（通常）
- [ ] 音声書庫（該当データがある場合）
